# Инструкция для агента — origg.in

Кратко: репозиторий — Nuxt 3 приложение (Nitro), бэкенд через `server/api/*`, Drizzle ORM + LibSQL/Turso, аутентификация и токены реализованы в `server/api/auth`.


**Как запускать проект (локально)**
- Установить зависимости: `npm install`
- Запустить dev-сервер: `npm run dev`
- Сборка: `npm run build`, предпросмотр: `npm run preview`

**Команды DB / миграции**
- Генерация/управление Drizzle: `npm run db:generate`, `npm run db:migrate`, `npm run db:push`, `npm run db:studio`.
- Переменные окружения для БД: `TURSO_DATABASE_URL`, `TURSO_AUTH_TOKEN`. Локально можно использовать sqlite-файл, но конфигурация рассчитана на Turso/LibSQL.

**Ключевые файлы и папки**
- `app/` — корневые компоненты и шаблон приложения.
- `pages/` — страницы Nuxt (index, login, register, cabinet и т.д.).
- `server/api/` — backend-эндпоинты (auth, user и т.д.).
- `server/database/schema.ts` — схема Drizzle (таблицы `users`, `tokens`, `admins`).
- `server/plugins/` — плагины для DB/Turso.
- `composables/` — клиентские хуки (`useApi.ts`, `useUserStore.ts`).

**Аутентификация и безопасность**
- Пароли хранятся с `bcrypt` (проверь `server/api/auth/*`).
- Токены и типы описаны в `schema.ts` (TokenType).
- Никогда не коммить секреты/файлы окружения.
- Перед выполнением миграций на проде — спросить владельца.

**Стиль работы агента — правила и ограничения**
- Действия, которые агент может выполнить автоматически:
  - Поправить код, отформатировать и прогнать локальные скрипты (`npm run dev`/`build`).
  - Создать/обновить миграции Drizzle (`npm run db:generate`) и подготовить изменения.
  - Добавить/обновить небольшие документационные файлы.
- Действия, требующие подтверждения пользователя:
  - Запуск миграций на боевой базе или пуш миграций в удалённый репозиторий.
  - Изменения, затрагивающие секреты/переменные окружения.
  - Удаление таблиц или других разрушительных изменений в БД.

**Что проверять при изменениях**
- Запустить `npm run dev` и убедиться, что сервер стартует без ошибок.
- Проверить затронутые API-роуты через простой HTTP-запрос (curl/postman).
- Для изменений в схеме — сгенерировать миграцию и показать дифф.


**Контакты и заметки**
- Файлы конфигурации: `nuxt.config.ts` (содержит runtimeConfig и ссылки на переменные окружения).
- Скрипты Drizzle указаны в `package.json`.
